npm

🖥️ Node.js|💰 免费|📅 2024年12月
包管理前端Node.js

为什么选择 npm

Node.js 自带的包管理器，也是全球最大的软件注册表。做前端开发，几乎每天都要和 npm 打交道——安装依赖、更新包、发布自己的包。

虽然现在有 yarn、pnpm 等替代品，但 npm 依然是事实标准，很多工具默认都支持 npm。

常用命令

⚡ 日常必用

  • npm install — 安装 package.json 里的依赖
  • npm install xxx — 安装某个包并保存到依赖
  • npm install -D xxx — 安装到开发依赖
  • npm update — 更新所有包
  • npm run dev — 运行 package.json 里的脚本
  • npm publish — 发布自己的包到 npm

✅ 优点

  • 生态最大，包最全
  • Node.js 内置，无需安装
  • npx 方便运行临时命令
  • 团队协作无障碍
  • npm scripts 很好用

❌ 缺点

  • 大型项目安装慢
  • node_modules 太占空间
  • 幽灵依赖问题
  • npm install 有时锁版本不一致

适合人群

所有 前端开发者 和 Node.js 开发者。不管你用 Vue/React 还是纯 Node 后端，都离不开 npm。

总结

前端开发的基础设施，虽然有些小问题但瑕不掩瑜。生态优势是其他包管理器短期内无法超越的。
